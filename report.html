Report

  1. The web service is not a to-do list manager.
  2. The web service uses a relational database written with MYSQL DDL.
  3. The web service uses express to deliver content.
  4. The web service is available on https://project2.shoresabroad.com:8443/.
  5. the web service's endpoints, and parameters are properly named.
  6. The web service sends and recieves complex data as JSON.
  7. The full spectrum of crud operations are supported for all tables.
  8. The database is only accessible from inside the droplet.
  9. The service is using an Nginx server.
  10. The service supports cross-origin resource sharing and has an options endpoint.
  11. The service is managed by pm2.
  12. The service is on a git repository and shared with twodee.
  13. The DDL is included in the repository, but the credentials for the server are not.
  14. My git repository has commits starting a few days ago until now.
  15. This service responds to https://project2.shoresabroad.com:8443/report.html with this page

Additional notes:

This service is built a part of a project for CS 374: Databases. This service accesses and manipulates a database that is in 3NF. I have included most of the design specifications and justifications at the following links:

  • ERD: https://project2.shoresabroad.com:8443/pdf/ERD.pdf
  • Data Dictionary: https://project2.shoresabroad.com:8443/pdf/Data Dictionary.pdf
  • Data Dependency Diagram: https://project2.shoresabroad.com:8443/pdf/Data Dependency Diagram.pdf
The service aims to reduce server time processing due to the target volume of requests, meaning that post, edit, and delete requests will only return status 204 and Success as the json response. Get requests will return formatted json data that accurately represents the format of the data types of the respective fields in the request. This service is the full suite of queries; however, for the client application, most of these endpoints will be unavailable due to security and data privacy risks.
